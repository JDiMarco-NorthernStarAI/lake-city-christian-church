REPLIT BUILD PROMPT: Lake City Christian Church Website
PROJECT OVERVIEW
Build a modern, full-stack church website for Lake City Christian Church (LC3 / LCCC) located at 6717 Fry Road, Middleburg Heights, OH. The website must include a built-in admin CMS so non-technical church staff can edit all content (text, images, YouTube videos, events) through a visual UI — no code editing required. The church currently uses a Wix site at lakecitycc.com and wants to move to a custom, modern build.
Mission Statement: "Lake City Christian Church exists to connect people to a life-changing relationship with Jesus."

TECH STACK

Frontend: React (Next.js preferred) with Tailwind CSS
Backend: Node.js with Express or Next.js API routes
Database: SQLite (via Prisma or better-sqlite3) for content, users, events, and sermons — lightweight, no external DB needed
Auth: Session-based auth with bcrypt password hashing. Admin login + role-based access (Admin = full access, Editor = assigned sections only)
File Uploads: Local file storage in /public/uploads/ with image upload UI in admin panel
Deployment: Single Replit project, runs with one command (npm run dev)

Typography

Headings: Bold, modern sans-serif — use Inter (800 weight) or Montserrat (700-800). Large sizes. Minimal words.
Body: Inter 400/500 at 16-18px base size.
Script/Logo Font (optional accent): A cursive font similar to what appears in the Lake City logo wordmark can be used sparingly for decorative headings — consider Playfair Display italic or Dancing Script for the "Lake City" text treatment.
DESIGN PHILOSOPHY: Short, clear directives. Less verbiage. White space is an advantage. Bolder fonts, fewer words. Fresh and clean.

ADMIN CMS REQUIREMENTS
Login & Authentication

Login page at /admin with username/password
Default admin account: username admin, password lakecity2024 (prompt user to change on first login)
Session-based auth with secure cookies
Logout button in admin nav

Role-Based Permissions
Two roles:

Admin — Can do everything: edit all pages, manage users, upload images, manage events/sermons
Editor — Can only edit sections they're assigned to (e.g., "Events Editor" can only manage announcements, "Kids Ministry Editor" can only edit the Kids page)

Admin can create new users and assign roles from the Users management page.
Admin Dashboard
Clean, simple dashboard with sidebar navigation:

Pages — List of all pages; click to edit
Sermons — Add/edit/delete sermon entries (YouTube URL + title + date + optional series name)
Events — Add/edit/delete announcements/events (title + date + image + body)
Team — Add/edit/remove leadership team members (name, role, bio, photo)
Media — Image upload library; browse/delete uploaded images
Users — Manage admin/editor accounts
Settings — Church name, address, service times, social links, giving URL

Content Editing UI
For each page, the admin sees:

The page broken into editable sections (hero text, body text, images, etc.)
Each section has a simple form: text fields for headings/body, image upload button for photos, URL field for YouTube videos
WYSIWYG is NOT required — simple text inputs and textareas are fine. The goal is simplicity.
Preview button to see changes before publishing
Save/Publish button

Image Handling

Admin can upload images (JPG, PNG, WebP) up to 5MB
Images auto-resize to max 1920px wide on upload
Uploaded images stored in /public/uploads/
Admin can choose to apply B&W filter or keep color per image

Breakpoints: mobile (<768px), tablet (768-1024px), desktop (>1024px)

IMPORTANT DESIGN NOTES FROM THE CHURCH

"Streamlined — limited click-throughs for people to get what they need without having to dig for information."


"Fresh and clean: white space can work as an advantage with bolder fonts and less verbiage. Short, clear directives."


"Engaging/approachable. We want a few photos of our church/people but they do not have to be featured on every page or within every prompt."


"We've leaned a lot towards black and white photos."


"The watermark has the Lake City blue color we use more often."

Translation for development:

Every page should have ONE clear purpose and ONE clear call-to-action
Use generous padding/margins (40-80px between sections)
Headlines should be punchy: 3-7 words max when possible
Body text: 2-3 sentences per block max
Photos are accent, not filler — use them intentionally
The blue gradient logo is the only color pop against the B&W photo aesthetic
Buttons should use the blue gradient for primary actions, outlined/ghost for secondary
