Build a complete backend system for this church app with user authentication, role-based access, event/signup management, and a dynamic form builder. This backend must be designed as a REST API that will serve a web app, iOS app, and Android app. Here are the requirements:

## ARCHITECTURE REQUIREMENTS FOR MULTI-PLATFORM SUPPORT

### API Design Principles
- Build a fully RESTful API that is platform-agnostic (no server-side rendering dependencies)
- All endpoints return JSON responses with consistent structure: { success: boolean, data: object|array, error: string|null, meta: object }
- Include pagination metadata on all list endpoints: { page, limit, total, totalPages }
- Use proper HTTP status codes (200, 201, 400, 401, 403, 404, 500)
- All dates/times in ISO 8601 format (UTC) - clients handle timezone conversion
- Version the API with /api/v1/ prefix for future compatibility

### Authentication for Mobile
- Implement JWT access tokens (short-lived, 15 minutes) + refresh tokens (long-lived, 30 days)
- Store refresh tokens in database with device_id, device_type (web|ios|android), and last_used timestamp
- POST /api/v1/auth/refresh - Exchange refresh token for new access token
- POST /api/v1/auth/logout-all-devices - Invalidate all refresh tokens for user
- GET /api/v1/auth/devices - List all logged-in devices for user
- DELETE /api/v1/auth/devices/:deviceId - Log out specific device
- Support Apple Sign-In (Sign in with Apple) for iOS
- Support Google Sign-In for Android and web
- Include endpoints:
  - POST /api/v1/auth/social/apple - Authenticate with Apple ID token
  - POST /api/v1/auth/social/google - Authenticate with Google ID token

### Push Notifications Infrastructure
- Create device_tokens table:
  - id (primary key)
  - user_id (foreign key to Users)
  - token (string, required)
  - platform (enum: 'ios', 'android', 'web')
  - is_active (boolean, default true)
  - created_at
  - updated_at
- Endpoints:
  - POST /api/v1/notifications/register-device - Register push token
  - DELETE /api/v1/notifications/unregister-device - Remove push token
  - GET /api/v1/notifications - Get user's notification history
  - PUT /api/v1/notifications/:id/read - Mark notification as read
  - PUT /api/v1/notifications/read-all - Mark all as read
- Create notification_preferences table:
  - user_id (foreign key)
  - event_reminders (boolean, default true)
  - new_events (boolean, default true)
  - donation_receipts (boolean, default true)
  - form_responses (boolean, default true)
  - ministry_updates (boolean, default true)
- Structure notification sending to easily integrate with Firebase Cloud Messaging (FCM) for Android and Apple Push Notification Service (APNs) for iOS

### File Upload for Mobile
- Support multipart/form-data uploads for mobile compatibility
- Return file URLs that work across all platforms
- Generate thumbnails for images (store reference in database)
- Include file metadata in responses: { url, thumbnailUrl, mimeType, size, filename }
- Structure storage abstraction layer for easy migration to AWS S3:
  - Current: Local storage with /uploads/ directory
  - Future: S3 bucket with CloudFront CDN URLs

### Offline Support Considerations
- Include updated_at timestamps on all records for sync
- GET /api/v1/sync/changes?since=ISO_TIMESTAMP - Get all changes since timestamp for offline sync
- Use soft deletes with deleted_at column so mobile apps can sync deletions
- Return deleted record IDs in sync endpoint

## API ENDPOINTS

All endpoints prefixed with /api/v1/

### Authentication
- POST /auth/register - Create new user (require: name, email, phone; optional: full address info)
- POST /auth/login - Login with email/password, returns access_token + refresh_token
- POST /auth/refresh - Exchange refresh token for new access token
- POST /auth/logout - Logout current device (invalidate refresh token)
- POST /auth/logout-all - Logout all devices
- POST /auth/forgot-password - Send password reset email
- POST /auth/reset-password - Reset password with token
- POST /auth/social/apple - Authenticate with Apple Sign-In token
- POST /auth/social/google - Authenticate with Google Sign-In token
- GET /auth/me - Get current user profile
- PUT /auth/me - Update current user profile
- PUT /auth/me/password - Change password
- PUT /auth/me/photo - Upload profile photo
- GET /auth/devices - List all logged-in devices
- DELETE /auth/devices/:deviceId - Log out specific device

### Push Notifications

- GET /notifications - Get user's notifications (paginated)

- PUT /notifications/:id/read - Mark as read
- PUT /notifications/read-all - Mark all as read
- GET /notifications/preferences - Get notification preferences
